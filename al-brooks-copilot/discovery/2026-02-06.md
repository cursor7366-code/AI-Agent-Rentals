## New Discoveries

### üî• Aurora ‚Äî Multimodal Time Series Foundation Model
- **Source**: GitHub / HuggingFace
- **Category**: Regime Inference, Signal Extraction
- **Project**: [A]
- **What it does**: First pretrained *multimodal* time series foundation model. Combines text context (news, events, domain knowledge) with time series data for probabilistic forecasting via flow matching.
- **Why interesting**: This is a paradigm shift. Instead of just feeding OHLC bars, you can inject context like "Fed announcement today, market likely news-driven" and the model uses that to inform predictions. Perfect for regime inference where context matters.
- **How you'd use it**: Feed OHLC bars + text context describing market conditions ‚Üí get regime probabilities with domain knowledge baked in. Could replace regime_inference.py entirely or augment TimesFM.
- **Maturity check**: ICLR 2026 paper (peer-reviewed), `pip install aurora-model`, active development
- **Initial verdict**: **HIGH** ‚Äî Solves the "how do I inject domain knowledge into regime inference" problem
- **Link**: https://github.com/DecisionIntelligence/Aurora

### Salesforce MoiraiAgent ‚Äî Expert Selection Tool
- **Source**: HuggingFace
- **Category**: Regime Inference, Ensemble
- **Project**: [A]
- **What it does**: Meta-model that selects the best prediction from multiple forecasting models. "Top-performing forecasters exhibit unique advantages when applied to time series from specific domains or with particular patterns."
- **Why interesting**: Instead of betting on ONE time series model, run several (TimesFM, Chronos, Aurora) and let MoiraiAgent pick the best prediction for each situation. Automatic ensemble without manual rules.
- **How you'd use it**: Run TimesFM + Chronos + Aurora in parallel ‚Üí MoiraiAgent picks the winner. Covers edge cases where one model excels.
- **Maturity check**: 3B params, Salesforce research, new (8 days ago)
- **Initial verdict**: **MEDIUM** ‚Äî Interesting meta-approach, but need core models working first
- **Link**: https://huggingface.co/Salesforce/moirai-agent

### LangChain 1.2.x ‚Äî Dynamic Tool Registration
- **Source**: GitHub Releases
- **Category**: Agent Architecture
- **Project**: [A]
- **What it does**: New middleware system allows dynamic tool registration during agent runs. Tools can be added/modified based on context.
- **Why interesting**: For the decision engine, this means tools available to the agent can change based on market regime. In ranging mode ‚Üí different tools than trending mode.
- **Maturity check**: langchain-core 1.2.9, very active development
- **Initial verdict**: **LOW** ‚Äî Nice-to-have, not blocking anything
- **Link**: https://github.com/langchain-ai/langchain/releases

## Synthesis & Analysis

### üß† The Multimodal Insight
Aurora represents a paradigm shift: **time series + context**. Previously, regime inference meant feeding bars and hoping the model figured out the context. Now you can explicitly tell the model "this is a Fed day, expect news-driven behavior."

**Combination to try**: Aurora (with text context) + TimesFM 2.5 (pure time series) + MoiraiAgent (pick the best). This creates a robust ensemble where:
- TimesFM handles pure pattern recognition
- Aurora handles context-aware predictions
- MoiraiAgent arbitrates when they disagree
